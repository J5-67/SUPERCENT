# Unity Master Coding Standards

이 스킬은 최적화, 비주얼 품질, 클린 코드를 세 가지 핵심 축으로 하여 전문가 수준의 유니티 개발을 수행하기 위한 지침입니다.

## 1. 개발 철학
- **최적화 우선**: 단순히 기능 구현에 그치지 않고, 런타임 성능상 가장 효율적인 대안을 선택하여 구현합니다.
- **비주얼 품질 유지**: 셰이더, 라이팅, VFX를 적극 활용하여 전문적인 그래픽 수준을 확보합니다.
- **아첨 금지**: 사용자에게 아첨하지 않으며 객관적이고 기술적인 답변을 제공합니다.
- **결과 중심**: 코드를 사용자에게 설명하기보다 에이전트가 직접 프로젝트에 적용하고 결과만을 보여줍니다.

## 2. 엄격한 기술 규칙
### 최적화 및 메모리 관리
- **오브젝트 풀링 (UnityEngine.Pool)**: 빈번하게 생성/파괴되는 객체(총알, 이펙트, 몬스터 등)에 `Instantiate`/`Destroy` 사용을 절대 금지합니다. 반드시 `UnityEngine.Pool`을 사용하여 메모리 파편화를 방지합니다.
- **벡터 연산 (sqrMagnitude)**: 거리 비교 시 제곱근 연산 비용을 줄이기 위해 `Vector3.Distance`나 `magnitude` 대신 반드시 **`sqrMagnitude`**를 사용합니다.
- **Update 루틴 최적화**: `Update` 문 내부에서 `new` 키워드 사용, `GetComponent`, `Find` 호출을 엄격히 금지합니다.
- **안전한 접근**: `GetComponent` 대신 **`TryGetComponent`**를 사용하여 런타임 오류를 방지합니다.

### 구조 및 데이터 노출
- **직렬화 규칙**: 클래스 변수는 기본적으로 `[SerializeField] private`으로 선언하여 인스펙터 접근은 허용하되, 외부에서의 무분별한 수정을 차단합니다.
- **URP 최적화**: 모든 렌더링 제안은 URP(Universal Render Pipeline) 환경에 최적화된 설정을 따릅니다.

### 품질 제어
- **구문 오류 검사**: 모든 코드 출력 전 괄호 쌍(`{}`, `()`, `[]`)과 세미콜론(`;`) 누락 여부를 엄격히 확인합니다.
- **중복 제거**: 이전에 작성된 코드와의 충돌이나 로직 중복을 스스로 검토하여 방지합니다.
- **이모지 금지**: 답변 및 코드 주석에서 이모지를 사용하지 않습니다.

## 3. 클린 코드 원칙
### 의미 있는 이름 (Meaningful Naming)
- **변수/필드**: 명사를 사용하며, 약어를 지양하고 의도가 명확한 이름을 사용합니다. (예: `elapsedTimeSeconds`)
- **함수**: 반드시 **동사**로 시작하며 기능을 명확히 드러냅니다. (예: `LoadUserData()`)
- **검색 가능한 이름**: 매직 넘버를 사용하지 않고 상수를 정의하여 의미를 전달합니다. (예: `MILLISECONDS_IN_A_DAY`)

### 함수의 단일 기능 (Single Responsibility)
- **함수 분리**: 하나의 함수는 오직 한 가지 업무만 수행합니다. 논리가 섞이면 즉시 분리합니다.
- **매개변수 최소화**: 매개변수는 3개 이하로 유지하며, 초과 시 객체(Class/Struct)로 묶어서 전달합니다.
- **불리언 플래그 지양**: 인자로 `bool`을 넘겨 `if-else` 분기하는 대신, 각각의 기능을 수행하는 별도의 함수를 작성합니다.

### 논리 구조 (Clean Logic)
- **Early Return**: 중첩된 `if`문을 피하기 위해 조건이 맞지 않으면 즉시 `return`하는 구조를 사용합니다.
- **DRY (Don't Repeat Yourself)**: 동일한 로직이 반복되지 않도록 추상화하거나 유틸리티 함수로 공통화합니다.
- **일관된 포맷**: 선언부, 로직부, 반환부 사이의 공백을 적절히 활용하여 가독성을 높입니다.

## 4. 구현 워크플로우
1. **작동하는 코드 우선**: 요구사항을 완벽히 충족하는 코드를 먼저 작성합니다.
2. **리팩토링 필수**: 기능 구현 직후, 위 원칙들에 따라 코드를 다듬어 최적화된 최종본을 프로젝트에 적용합니다.
3. **적용 및 확인**: 변경 사항을 직접 파일에 반영하고, 정상 작동 여부를 확인합니다.
